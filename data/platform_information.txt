# 시스템 소개

## Vision system 구성

1. 시스템의 주요 특징
    1. 검사 관련 기능을 모듈화 하여, 사용자 개별 설정 및 변경 적용 가능
    2. Windows 기반 S/W 으로 구현되어 추가 및 확장 용이
    3. 고화질 영상 획득을 통해, 안정적인 측정이 가능
    4. 필요에 따라 검사 조건 변경, 카메라 추가 등이 용이함
    5. 2개 이상의 검사 항목에 대해 복수의 검사 처리가 가능
2. 시스템 일반 사항
    1. 시스템 명: 한화큐셀 통합 비전 검사 시스템
    2. 검사기 일반 사양
        - 카메라와 조명을 사용하여 외관 검사 및 위치 보정 값 계산을 수행한다.
    3. 검사기 구성
        - Camera와 컨트롤러가 포함된 Led 조명으로 구성됨
        [시스템소개_001]

# 소프트웨어 소개

## 1. 공통 UI

- [공통UI_001] 은 Vision S/W 실행 시 생성되는 화면으로 Main 화면에 해당한다.
- **①**과 **②**는 공통으로 사용되는 UI로 창 전환을 하여도 변화가 없다
1. 상단 공통 UI
    1. [공통UI_002] : 회사 로고 표시
    2. [공통UI_003] : Vision S/W 와 통신 연결 상태 표시
        1. PC 연결 상태
            1. 두 가지의 상태가 표시되며 내용은 다음과 같다.
                - [공통UI_004] : Master / Slave PC 와 연결된 상태
                - [공통UI_005] : Master / Slave PC 와 연결이 되지 않은 상태
        2. Camera 연결 상태
            1. 세 가지의 상태로 표시되며 내용은 다음과 같다.
                - [공통UI_006] : Camera 와 연결된 상태
                - [공통UI_007] : 가상의 Camera (Simulation) 와 연결된 상태 ( 해당 상태일 경우 실제 카메라로 영상 취득은 되지 않음)
                - [공통UI_008] : Camera 와 연결이 해제된 상태
        3. PLC 연결 상태
            1. 세 가지의 상태로 표시되며 내용은 다음과 같다.
                - [공통UI_009] : PLC 와 연결된 상태
                - [공통UI_010] : 가상의 PLC (Simulation) 와 연결된 상태 ( 해당 상태일 경우 PLC 와의 통신은 되지 않는다. )
                - [공통UI_011] : PLC 와 연결이 해제된 상태
        4. [공통UI_012] : Vision PC 상태를 나타낸다
            1. CPU : Vision PC 의 CPU 사용량 표시
            2. RAM : Vision PC 의 RAM 사용량 표시
            3. SSD : Vision PC 의 C 드라이브 사용량 표시
            4. HDD : Vision PC 의 D 드라이브 사용량 표시
        5. [공통UI_013] : Vision S/W 현재 version 표시
        6. [공통UI_014] : Manual 호출 버튼
        7. [공통UI_015] : 현재 날짜와 시간
2. 우측 공통 UI : VIsion S/W 의 기본적인 기능을 사용하기 위한 메뉴들이 구성되어 있음
    1. [공통UI_016] : 현재 모델과 공정 이름이 표시
    2. [공통UI_017] : Navigation Bar : Vision S/W 화면 전환 기능
        - [공통UI_018] : Main 화면으로 전환
        - [공통UI_019] : Calibration 화면으로 전환
        - [공통UI_020] : Model 화면으로 전환
        - [공통UI_021] : Monitor 화면으로 전환
        - [공통UI_022] : Setup 화면으로 전환
        - [공통UI_023] : Alarm 화면으로 전환
        - [공통UI_024] : History 화면으로 전환
    3. [공통UI_025] : Vision S/W 구동 상태를 전환. 마우스 클릭으로 상태 변경
        - Manual : 모델, 설정에서 파라미터 변경과 Camera Live 영상 취득을 할 수 있다. 설비 구동 중 Vision S/W가 MANUAL이라면 PLC 신호를 받지 않는다
        - Auto : 설비 구동 시 Vision 검사를 진행하며 PLC 로 결과 데이터를 전달한다. 파라미터 변경 및 Camera Live 영상 취득을 할 수 없다
    4. [공통UI_026] : Batch 기능을 사용할 수 있는 창을 생성
    5. [공통UI_027] : 결과 데이터를 볼 수 있는 Result 창 생성
    6. [공통UI_028] : Simulation 을 위한 가상의 PLC 창을 생성
    7. [공통UI_029] : Vision S/W 의 Data 및 Sequence 를 초기화
    8. [공통UI_030] : Vision S/W 종료

## 2. Main UI

- [MainUI_001] 은 우측 공통 UI 에서 Main 버튼을 눌렀을 때 전환되는 화면
- Vision S/W 에서 검사된 이미지 및 결과 데이터를 볼 수 있음
1. Display page tab : Vision S/W 에서 촬영된 제품 영상이 표시되며 결함이 있을 경우 결함 위치를 표시
    - [MainUI_002] : 하나의 화면에 여러 개의 Display를 보여줄 경우 Display 안의 Image는 크기가 작고 여러개의 Display로 가시성이 떨어진다. 이러한 문제 해결을 위해 여러 개의 Page를 만들고 해당 Page로 이동할 수 있는 것이 Display Page Tab 이다.
    - [MainUI_003]
        - [MainUI_004] : Display Page 분리 (Page 1)
        - [MainUI_005] : Display Page 분리 (Page 2)
        - [MainUI_006] : Display Page 분리 (Page 3)
2. 검사 데이터 출력 창 : Vision S/W 에서 검사한 결과 데이터가 출력되는 창
    - Unit (공정) 별로 Insp result 와 Align result 로 생성이 되며 마지막 두 개의 Tab 은 Inspection 과 Align 의 spec 정보로 출력됨
    - Unit 이름은 Model UI 의 Unit 의 title 이름으로 변경이 가능
    - [MainUI_007] : Insp result
        - 길이 및 외관 검사의 치수나 판정 데이터를 표시
        - Model, Start time (PLC start 데이터 받은 시간), Module ID, Result 항목은 고정이며, 이후 항목들은 Unit(공정) 별 검사 항목에 따라 검출된 데이터 값이 출력됨
        - 검사 결과가 NG 일 경우, Result 와 NG data 는 붉은색으로 표시됨
    - [MainUI_008] : Align result
        - 제품 Align 후 해당 Align 의 판정과 보정값을 표시. 판정은 Align spec 으로 결정되며, 판정이 NG 일 경우 빨간 글씨로 표기
        - Target X, Target Y, Target T : PLC 로 전달되는 Align 보정값 (Revision + Offset)
        - Revision X, Revision Y, Target T : Vision S/W Align Algorithm 으로 계산된 보정값
        - Offset X, Offset Y, Offset T : Align 위치를 미세 조정하기 위한 기구적 위치
    - [MainUI_009] : Inspection spec
        - 현재 적용된 측정 또는 외관 검사 항목의 spec 을 표시
        - Model 창에서 설정한 spec 으로 해당 spec 에 만족해야 OK 로 판정
        - Unit : 공정 항목 내에 측정 또는 외관 검사 Tool 항목 표시
        - Insp Name : 측정 또는 외관 검사 tool 의 검사 방법 표시
            - Limit : standard 기준 하한 spec 표시. 해당 값 미만의 값은 NG 판정
        - Standard : 기준 spec 표시
            - Limit : standard 기준 상한 spec 표시. 해당 값 초과의 값은 NG 판정
    - [MainUI_010] : Align spec
        - 현재 적용된 Align 항목의 spec 표시
        - Model 창에서 설정한 spec 으로 해당 spec 에 만족해야 OK 로 판정
        - Unit : 공정 항목 내에 align tool 항목 표시
        - Align name : Aligner tool 의 title 표시
            - Limit X, - Limit Y, - Limit T : PLC 로 전달되는 align 보정값 Target X, Target Y, Target T 의 하한 spec 표시. 해당 값 미만의 값은 NG 판정
            - Limit X, + Limit Y, + Limit T : PLC 로 전달되는 align 보정값 Taget X, Taget Y, Taget T 의 상한 spec 표시. 해당 값 초과의 값은 NG 판정

## 3. Calibration UI [CalibrationUI_001]

- 영상의 1Pixel 당 실제 거리를 계산하여 픽셀당 거리(mm)의 비율을 구할 수 있다.
- Align 보정값 계산 시 보정하는 로봇의 회전 중심을 구할 때 사용되는 화면이다.
- Calibration 화면에서는 영상을 출력하여 거리 측정 등의 작업을 할 수 있다.
- 여러 개의 카메라에서 영상을 동시에 출력할 수 있다.
- 필요에 따라 보조선을 긋거나, 조명 밝기를 변경하는 작업도 가능하다.
1. Display
[CalibrationUI_002] : Camera grab 또는 Live 영상이 출력되는 Display
2. Display count
[CalibrationUI_003]
    - Display 수량을 증가시키며 Camera의 실시간 영상을 볼 수 있다.
    - Display 별 영상에서 길이 측정을 진행 할 수 있다. (단, Resolution Data가 있어야 정확한 실측이 가능하다)
    - [CalibrationUI_004] : Display 수량을 변경할 수 있다. 최소 개수는 1이다.
    - [CalibrationUI_005] : 생성한 Display로 출력할 Camera를 선택한다. 실시간 영상을 보기 위해서 LiveUse가 체크되어 있어야 한다.
    - [CalibrationUI_006] : LiveUse가 체크되어 있는 Camera들의 실시간 영상을 Display에 출력한다. Mouse Click으로 활성화가 되며 [CalibrationUI_007] 그림과 같이 변경된다. 사용을 안 할 경우 다시 Mouse Click을 하면 원상태로 돌아온다.
    - [CalibrationUI_008] : Calibration Data가 있을 경우 사용 가능하다. Mouse Click으로 활성화가 되며 [CalibrationUI_009] 그림과 같이 변경된다. 활성화 상태에서 Display에서 측정 하고자 하는 위치를 Mouse Click & Drag로 측정을 할 수 있다. Drag 시 Display엔 삼각형이 그려지며 삼각형의 세 개의 꼭지점으로 길이를 측정한다.
    - [CalibrationUI_010] : 위의 MEASURE 기능 사용 시 측정된 길이의 데이터가 출력된다.
3. 조명 컨트롤러
[CalibrationUI_011]
    - 조명 컨트롤러와 조명 컨트롤러의 채널을 선택 후 조명 값을 변경할 수 있다.
    - [CalibrationUI_012] : 두 종류의 조명 컨트롤러가 있으며 사용을 위해선 SETUP UI에서 조명 컨트롤러의 사용을 먼저 설정해야 한다.
    - [CalibrationUI_013] : 조명 컨트롤러 종류에 따라 채널의 개수가 다르다. 변경하고자 하는 채널을 선택할 수 있다.
    - [CalibrationUI_014] : 파란색 바를 Mouse Click & Drag 로 움직일 수 있으며 움직인 양 만큼 조명 값이 변경 된다.
    - [CalibrationUI_015] : 값을 입력한 수치로 조명 값이 변경 된다. 위의 파란색 바를 움직여도 해당 값이 변경 된다. 최소 0에서 최대 255 까지 의 단계로 이루어져 있다.
4. 수동 calibration
[CalibrationUI_016]
    - Align 을 하는 공정에서 보정을 하는 Robot의 회전 중심을 구할 때 사용된다. (수동 Calibration)
    - 수동 Calibration 을 하기 위해 Model 창에서 Camera의 Resolution Data 를 구해야 하며 Detect의 Mark Index도 설정해야 한다.
    - [CalibrationUI_017] : 수동 Calibration을 할 수 있는 Camera 항목이 Tab 으로 되어있다.
    - [CalibrationUI_018] : Camera 를 TargetID 로 가지고 있는 Detect 항목이 있다. Calibration 하고자 하는 Detect 을 선택할 수 있으며, 이미 Calibration 을 했다면 Calibration Data 를 볼 수 있다.
    - [CalibrationUI_019] : SELECTED DETECT에서 선택한 Detect의 Align Type 정보를 볼 수 있다. 해당 설정은 Model 창에서 변경 가능하다.
    - [CalibrationUI_020] : SELECTED DETECT에서 선택한 Detect의 Mark Index 정보를 볼 수 있다. 해당 설정은 Model 창에서 변경 가능하다.
    - [CalibrationUI_021] : Calibration 을 진행할 이미지의 가로 Pixel 크기를 입력한다. Image Load를 할 경우 또는 Calibration Data가 있을 경우 IMAGE WIDTH 를 볼 수 있다.
    - [CalibrationUI_022] : Calibration 을 진행할 이미지의 세로 Pixel 크기를 입력한다. Image Load를 할 경우 또는 Calibration Data가 있을 경우 IMAGE HEIGHT 를 볼 수 있다.
    - [CalibrationUI_023] : Calibration 을 할 제품의 실제 가로 길이를 입력한다. Calibration Data가 있을 경우 PRODUCT WIDTH 를 볼 수 있다.
    - [CalibrationUI_024] : Calibration 을 할 제품의 실제 세로 길이를 입력한다. Calibration Data가 있을 경우 PRODUCT HEIGHT 를 볼 수 있다.
    - [CalibrationUI_025] : Align 보정을 하는 Robot의 회전 중심 좌표 중 X축을 입력 값만큼 이동 시킨다. 이동이 필요 없을 경우 0을 입력한다. 입력 데이터의 단위는 mm이다.
    - [CalibrationUI_026] : Align 보정을 하는 Robot의 회전 중심 좌표 중 Y축을 입력 값만큼 이동 시킨다. 이동이 필요 없을 경우 0을 입력한다. 입력 데이터의 단위는 mm이다.
    - [CalibrationUI_027] : Camera의 Live 영상이 아닌 저장되어 있는 이미지로 Calibration 을 진행할 경우 선택한 이미지를 불러올 수 있다.
    - [CalibrationUI_028] : 직교 좌표계 또는 Robot의 회전 중심 중 구하고자 하는 Calibration Data 를 선택하여 구할 수 있는 창을 생성한다.
        - **Manual cal 을 선택한 경우**
        [CalibrationUI_029]
        - [CalibrationUI_030] : Create Coordinate(직교 좌표계) 또는 Theta Calibration (Robot의 회전 중심) 중 구하고자 하는 항목을 선택한다.
            - [CalibrationUI_031] : 체크 시 Display에 포인트가 생기며 Mouse Click 으로 포인트를 이동 시킬 수 있다.
            - [CalibrationUI_032] : Use Graphic Point 체크 시 생성된 포인트의 Pixel X값을 출력한다.
            - [CalibrationUI_033] : Use Graphic Point 체크 시 생성된 포인트의 Pixel Y값을 출력한다.
        - **Theta calibraiton 을 선택한 경우**
        [CalibrationUI_034]
            - [CalibrationUI_035] : 체크 시 Display에 포인트가 생기며 Mouse Click 으로 포인트를 이동 시킬 수 있다.
            - [CalibrationUI_036] : Use Graphic Point 체크 시 생성되는 두 개의 포인트 중 빨간 포인트의 위치 이동 시 선택한다.
            - [CalibrationUI_037] : Use Graphic Point 체크 시 생성되는 두 개의 포인트 중 파란 포인트의 위치 이동 시 선택한다.
            - [CalibrationUI_038] : Use Graphic Point 체크 시 생성된 빨간 포인트의 Pixel X값을 출력한다.
            - [CalibrationUI_039] : Use Graphic Point 체크 시 생성된 빨간 포인트의 Pixel Y값을 출력한다.
            - [CalibrationUI_040] : Use Graphic Point 체크 시 생성된 파란 포인트의 Pixel X값을 출력한다.
            - [CalibrationUI_041] : Use Graphic Point 체크 시 생성된 파란 포인트의 Pixel Y값을 출력한다.
            - [CalibrationUI_042] : -Point와 +Point의 Theta 차이를 입력한다.
        - **Move Point 항목 [CalibrationUI_043] : Display에 생성된 Point 를 미세 조정하여 위치를 변경할 경우 사용된다.**
            - [CalibrationUI_044] : Click 시 Display 생성된 Point를 현재 위치에서 위로 이동 시킨다.
            - [CalibrationUI_045] : Click 시 Display 생성된 Point를 현재 위치에서 아래로 이동 시킨다.
            - [CalibrationUI_046] : Click 시 Display 생성된 Point를 현재 위치에서 좌측으로 이동 시킨다.
            - [CalibrationUI_047] : Click 시 Display 생성된 Point를 현재 위치에서 우측으로 이동 시킨다.
            - [CalibrationUI_048] : Point의 이동량을 1Pixel로 움직일지 사용자가 지정한 만큼 움직일 지 선택할 수 있다.
            - [CalibrationUI_049] : Display에 생성된 Point를 입력된 값만큼 회전시킬 수 있다.

2. Model Recipe
    1. Unit
        - Model Device에서 Model(Recipe) 작업 상세 설명
        - Model은 크게 Common, Cim, Unit으로 구분되어 있다.
            - Common : Vision S/W 구동 시 공통으로 사용하는 데이터들을 다룬다
            - Cim : Vision Data를 상위 보고할 경우(CIM 통신) 사용된다.
            - Unit : 검사 공정 별로 Unit이 생성되며 실제 검사를 하기 위한 장치들로 구성된다.
        - Unit은 Camera와 Image, Calibration, Result를 가질 수 있다.
            - Camera : 해당 검사 공정의 Camera 기능을 수행한다.
            Camera는 Indicator(지시툴),Light(조명 컨트롤러),Detect Device(특수 목적),Image Processing Tool을를 가질 수 있다.
            - Image : Camera에서 취득된 이미지를 전달받아 검사를 수행한다. Image는 Detect, Align, Inspection을 가질 수 있다.
                - Detect : 점,선,면등을 검출하는 검출기이다. Detect에는 Image Processing 기능을 넣을 수 있다.
                - Align : 공정에서 보정값이 필요할 경우 사용되며 Detect으로부터 좌표 데이터를 전달받아 보정값을 계산한다. 또한 검사 위치가 동적으로 변경될 경우 특정 위치로부터 검사 위치를 자동 보정 가능한 Fixture Align 기능도 포함되어 있다.
                - Inspection : 공정에서 측정 또는 외관 검사를 할 경우 사용되며 Detect으로부터 전달받은 검출 데이터를 조합하여 Spec에 맞는지 확인한다.
        - Calibration : Auto Calibration 진행 시 사용된다. (PLC와 통신을 통하여 Stage 또는 Robot이 위치 이동을 하며 Camera가 영상 취득 후 Resolution 및 회전 중심을 계산)
        - Result : 검사 완료 후 PLC로 데이터 전달 및 디스플레이 할 Align, Inspection의 데이터를 관리한다.
        - 작업 완료 후 반드시 SAVE 버튼을 눌러야 한다.

4. Detect
    - Detect을 추가할 Image를 선택 후 마우스 우측 버튼을 클릭하면 생성되는 메뉴에서 Add Detect을 클릭하면 Add Detect Device 창이 생성된다.
    - [ModelUI_205] : Detect을 등록 및 설정할 수 있는 창이다. 여러 개의 Detect을 한 번에 등록할 수 있으며 기본적인 설정을 진행할 수 있다.

Target ID는 영상을 전달받을 Camera를 설정하며 Count는 생성할 Detect의 설정 값이 같을 때 Count 만큼 Detect을 자동 생성한다. Title은 Model Device에 출력될 Detect의 이름이며 Processing은 Detect Tool 사용 전 Image를 Processing할 경우 설정한다.
